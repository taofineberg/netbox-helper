Platform Guide

This guide explains what each tab does, how data flows between source and destination NetBox servers, and how to read logs/errors/remediation when a sync or import fails.

Tab Map

TabPurposeKey Outcome
CSV ImportQueue CSV files and import into NetBox.Creates/updates site data via ordered import stages.
Server SyncCompare NetBox servers and sync selected differences.Item-level sync, site-level sync, and diff-driven updates.
ZabbixPlan and push Zabbix mapping/config from NetBox objects.Host groups, tags, inventory, interfaces, and assignment actions.
SettingsManage servers, users, and passwords.Controls which NetBox instances all tools can target.

High-Level Logic

1) Select source + destination server(s).
2) Choose branch strategy (Auto recommended).
3) Compare / plan to identify differences.
4) Execute sync/import in ordered sections.
5) Review:
   - Live progress
   - Created / Updated / Skipped / Errors
   - Logs, Errors, Remediation
6) Validate in destination branch before merge.

CSV Import

What It Does

  • Uploads one or more CSV files into a staging table.
  • Queues imports and runs them in background worker order.
  • Supports auto-branch targeting to avoid writing directly to main.
  • Shows live counters and detailed logs with error/remediation grouping.

Use this when onboarding or bulk-updating site/object data from prepared templates.

Server Sync

Modes

  • Object Compare: Git-style diff view per item, then sync source → destination or reverse.
  • Sync Full Site: Runs sections in import order (devices/modules/cables/IPAM etc.) with live per-type metrics.
  • Template Sync: Legacy item sync flow now hosted in this tab for continuity.

Branch-aware behavior is enforced; writes should target branch scopes, not main.

Zabbix

Planner + Execute

  • Pulls candidate NetBox devices and computes planned Zabbix actions.
  • Shows diff chunks before execution for safer changes.
  • Supports dry-run and execution paths with detailed outcome lines.
  • Includes options for host groups, tags, inventory, interfaces, and inheritance.

Logs, Errors, Remediation

How To Read Failures

  • Logs: full chronological run output.
  • Errors: concise grouped failures.
  • Remediation: actionable hints (missing refs, invalid values, duplicates, ordering issues).
  • For branch errors, confirm destination branch and header compatibility before retry.
